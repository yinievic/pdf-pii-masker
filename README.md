PDF 개인(신용)정보 마스킹 웹앱(ver 1.0)

이 웹앱은 PDF 내 개인 식별 정보(현재 설정: 주소 및 주민등록번호)를 사용자가 직접 검토하면서 마스킹하고, 마스킹 결과가 실제 반영된 이미지 기반 PDF를 다운로드할 수 있도록 하는 웹앱입니다. 원본 PDF를 직접 수정하지 않고, 각 페이지를 이미지로 렌더링한 뒤 선택된 마스킹 영역을 반영하여 새 PDF 사본을 생성하는 방식을 사용합니다.

주요 사용 흐름은 다음과 같습니다.:

사용자는 PDF 파일을 업로드하고 페이지별 미리보기를 확인합니다. 이후 필요에 따라 자동 개인정보 탐지를 실행하거나, 자동 탐지를 생략하고 직접 수동 마스킹만 수행할 수 있습니다. 자동 탐지를 실행하면 OCR 결과가 곧바로 확정되지 않고 마스킹 후보로 표시되며, 사용자는 페이지 위에서 필요한 후보는 유지하고 불필요한 후보는 해제할 수 있습니다. 자동 탐지 결과가 부족한 경우에는 누락 탐지 보완 기능을 실행할 수 있으며, 같은 페이지에서 수동 마스킹 박스를 추가해 자동 탐지 결과를 보완할 수 있습니다. 최종적으로 사용자가 선택한 자동 후보와 수동 마스킹 박스를 합산한 결과가 이미지 기반 PDF에 반영됩니다.

수동 마스킹은 PDF 페이지 위에서 드래그하여 박스를 생성하는 방식으로 동작하며, 박스 좌표는 canvas 기준으로 관리됩니다.
자동 탐지는 내부 OCR API를 통해 수행되며, Tesseract 및 poppler 기반 PDF 이미지 변환을 사용합니다. OCR 결과에서는 주민등록번호와 주소 등 사전에 정의된 탐지 규칙을 기준으로 후보를 생성합니다.
마스킹 데이터는 수동/자동 여부와 관계없이 공통 MaskBox 구조로 관리됩니다. 사용자가 유지한 자동 후보, 해제하지 않은 후보, 직접 추가한 수동 박스는 최종 적용 대상인 finalMasks로 산정됩니다. PDF 생성 단계에서는 이 finalMasks만 공통 다운로드 엔진에 전달됩니다. 따라서 PDF 생성 로직은 수동 마스킹, 자동 탐지, 자동 후보 해제, 수동 보완 여부에 따라 별도로 분기하지 않고 하나의 이미지 기반 PDF 생성 흐름으로 처리됩니다.
결과 PDF는 원본 PDF를 직접 편집하지 않고, 렌더링된 페이지 이미지 위에 마스킹 박스를 그린 뒤 새 PDF로 재생성합니다. 이 방식은 일반적인 “이미지로 인쇄”와 유사한 결과를 만들며, 원본 텍스트 레이어가 결과 파일에 남지 않도록 하는 것을 목표로 합니다.
마스킹 색상은 흰색과 검정 중 선택할 수 있으며, 현재 기본 마스킹 색상은 흰색입니다.

저장 정책은 문서 보안의 필요성을 충족합니다. 원본 PDF와 결과 PDF는 서버에 저장하지 않습니다. 다운로드는 브라우저 Object URL 기반의 세션성 링크로 제공되며, 새 다운로드 생성, 수동 마스킹 추가, 새 파일 업로드, 파일 제거, 화면 종료 등 작업 상태가 변경될 때 기존 링크를 해제합니다. OCR 처리가 필요한 경우에도 요청별 임시 디렉터리를 사용하고, 처리 종료 후 임시 파일을 삭제하는 구조를 전제로 합니다.

PDF Personal Information Masking Web App (ver. 1.0)

This web app allows users to review and mask personal identifying information in PDF files. The current detection settings cover addresses and resident registration numbers. After masking, the app generates a downloadable image-based PDF in which the masking is actually applied. The original PDF is not directly modified. Instead, each page is rendered as an image, the selected masking areas are applied on top of that rendered page, and a new PDF copy is generated.

Main workflow:

The user uploads a PDF file and reviews the page-by-page preview. The user may then run automatic personal information detection, or skip automatic detection and perform manual masking only. When automatic detection is executed, OCR results are not finalized immediately. Instead, they are displayed as masking candidates. The user can keep the necessary candidates and exclude unnecessary ones directly on the page. If the automatic detection result appears incomplete, the user may run the supplemental missing-detection function. The user can also add manual masking boxes on the same page to supplement the automatic detection result. The final image-based PDF is generated by combining the automatic candidates selected by the user and the manually added masking boxes.

Manual masking works by dragging on the PDF page to create a masking box. The box coordinates are managed based on the canvas coordinate system.
Automatic detection is performed through an internal OCR API. The OCR pipeline uses Tesseract and poppler-based PDF image conversion. Based on predefined detection rules, the OCR result generates candidates for items such as resident registration numbers and addresses.
Masking data is managed through a common MaskBox structure regardless of whether it was created manually or automatically. Automatic candidates kept by the user, candidates that have not been excluded, and manually added boxes are calculated as the final masking targets, referred to as finalMasks. During PDF generation, only these finalMasks are passed to the common download engine. Accordingly, the PDF generation logic is not separately branched depending on whether the masks are manual, automatic, excluded automatic candidates, or manual supplements. Instead, all cases are handled through a single image-based PDF generation flow.
The resulting PDF is not created by editing the original PDF directly. The app renders each page as an image, draws the masking boxes on top of the rendered image, and generates a new PDF. This approach produces a result similar to “print as image” and is intended to ensure that the original text layer does not remain in the output file.
The masking color can be selected as either white or black. The current default masking color is white.

The storage policy is designed to satisfy document security requirements. The original PDF and the resulting PDF are not stored on the server. Downloads are provided as session-based links using browser Object URLs. Existing links are revoked when the work state changes, such as when a new download is generated, manual masking is added, a new file is uploaded, a file is removed, or the screen is closed. When OCR processing is required, the system is designed to use a temporary directory for each request and delete temporary files after processing is completed.
